# Community Priorities Maps

Leaflet-based **Community Priorities** and **Cluster Priorities** maps for Baghlan-e-Jadid and Nawabad, Kunduz. Photo-backed FGD priorities and infrastructure layers from the Integrated Locations Database.

**Production (isolated):** [https://d1b6znwb7yuvt4.cloudfront.net](https://d1b6znwb7yuvt4.cloudfront.net)

| Map | URL |
|-----|-----|
| Full map (community priorities + ILD layers) | https://d1b6znwb7yuvt4.cloudfront.net/ |
| Cluster Priorities Only | https://d1b6znwb7yuvt4.cloudfront.net/cluster-priorities-map/map.htm |

## Repository structure

```text
frontend/
  community-priorities-src/   # Maintainable source — full map
  cluster-priorities-src/     # Cluster-only variant (shared app.js/styles.css)
  dist/                       # Packaged maps (generated by sync scripts)
deployed/
  cursor_v2_map_data/         # Generated JS bundles, icons, photo previews
  scripts/                    # Data generators (FGD photos, infrastructure Excel)
  Assets Needed/              # Local source assets (not committed)
backend/                      # Optional auth-only API (maps login)
sync-*.ps1                    # Package maps into frontend/dist
deploy-community-priorities-map-isolated-to-aws.ps1
deploy-maps-to-aws.ps1        # Generate → sync → images → isolated deploy → verify
```

## Run locally

```powershell
cd frontend
npm install
npm run sync:all-priorities
npm run dev
```

| Map | Local URL |
|-----|-----------|
| Community Priorities | http://localhost:5173/community-priorities-map/map.htm |
| Cluster Priorities | http://localhost:5173/cluster-priorities-map/map.htm |

For local community-map photos, copy previews after sync:

```powershell
Copy-Item -Recurse -Force deployed\cursor_v2_map_data\photo_previews frontend\dist\community-priorities-map\cursor_v2_map_data\photo_previews
```

## Regenerate map data

```powershell
cd deployed
npm install
npm run generate:data              # community priorities
npm run generate:infrastructure    # cluster / infrastructure priorities
```

Then re-sync:

```powershell
cd frontend
npm run sync:all-priorities
```

## Deploy to AWS (isolated)

Deploys to separate `community-priorities-map-*` S3 buckets and CloudFront. **Does not modify** the legacy main app at `d113s7v6pd04w6.cloudfront.net`.

```powershell
.\deploy-maps-to-aws.ps1
```

Or step by step:

```powershell
.\sync-community-priorities-map.ps1
.\sync-cluster-priorities-map.ps1
.\sync-priority-map-images-to-s3.ps1
.\deploy-community-priorities-map-isolated-to-aws.ps1
```

Verify:

```powershell
.\test-deployment-online.ps1
```

Optional CloudFront Basic Auth (isolated distributions only):

```powershell
.\enable-community-priorities-cloudfront-auth.ps1
```

## Auth API

Production maps use:

```text
https://tfqmwiadc8.execute-api.us-east-1.amazonaws.com
```

The `backend/` folder contains an **optional** auth-only Serverless service (`community-priorities-auth-api`). Deploy it only if you need a standalone auth endpoint — do **not** deploy it over the existing `community-profile-api` stack unless intentional.

```powershell
cd backend
cp .env.example .env   # set JWT_SECRET
npm install
npm run build
npm run deploy:dev
```
